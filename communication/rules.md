# Communication Rules

Rules for AI agents when composing emails, messages, and other communications.

## Core Communication Principles

### Clarity
- State the purpose early
- Use simple, direct language
- One main topic per message
- Be specific about requests
- Avoid ambiguity

### Professionalism
- Maintain appropriate formality
- Use proper grammar and spelling
- Respect the recipient's time
- Be courteous and respectful
- Proofread before sending

### Responsiveness
- Acknowledge receipt promptly
- Set expectations for response time
- Follow up appropriately
- Close communication loops

## Email Structure

### Subject Lines
- Clear and specific
- Action-oriented when appropriate
- Include relevant identifiers
- Keep under 50 characters

**Good Examples:**
- "Project Alpha: Design Review Needed by Friday"
- "Question about Q4 Budget Allocation"
- "Action Required: Update Your Contact Info"

**Avoid:**
- "Quick question"
- "Important"
- "Follow up"

### Email Format
```
Subject: [Clear, specific subject line]

Hi [Name],

[Opening: Purpose of the email in 1-2 sentences]

[Body: Details, context, and supporting information]

[Action items or next steps - clearly stated]

[Closing]

Best regards,
[Your name]
```

### Email Length
- **Brief updates**: 2-4 sentences
- **Standard messages**: 3-4 short paragraphs
- **Detailed explanations**: Use bullet points and sections
- **Complex topics**: Consider a meeting or document instead

## Types of Communication

### Request Emails
```
Subject: Request: [Specific Item/Action] by [Date]

Hi [Name],

I'm reaching out to request [specific item/action] for [reason/project].

Background: [Brief context if needed]

Specifically, I need:
- Item 1
- Item 2

Could you please provide this by [date]? If that timeline doesn't work, please let me know what's feasible.

Thank you for your help,
[Your name]
```

### Status Updates
```
Subject: Status Update: [Project/Task] - [Date]

Hi [Team/Name],

Here's the status update for [project/task]:

**Completed:**
- Item 1
- Item 2

**In Progress:**
- Item 3 (Expected: [date])

**Blockers:**
- Issue requiring attention

**Next Steps:**
- Planned action 1
- Planned action 2

Let me know if you have any questions.

Best,
[Your name]
```

### Follow-up Messages
```
Subject: Follow-up: [Original Subject]

Hi [Name],

I wanted to follow up on my [email/message] from [date] regarding [topic].

[Brief reminder of the request/question]

Please let me know if you need any additional information.

Thanks,
[Your name]
```

## Instant Messaging

### Best Practices
- Lead with the ask or key information
- Keep messages concise
- Use threads for related discussions
- Respect busy/DND status
- Don't expect immediate responses

### Format
- One idea per message
- Use formatting for readability
- Include context in first message
- Summarize long discussions

### When to Use
- Quick questions
- Informal coordination
- Time-sensitive matters
- Real-time collaboration

### When Not to Use
- Complex topics (use email or meeting)
- Sensitive information
- Things that need documentation
- When recipient is unavailable

## Meeting Communications

### Meeting Requests
- Clear purpose in title
- Detailed agenda in description
- Appropriate duration
- Consider time zones
- Include relevant links/materials

### Meeting Notes Template
```markdown
# Meeting: [Title]
**Date**: [Date]  
**Attendees**: [Names]

## Agenda
1. Topic 1
2. Topic 2

## Discussion
- Key point 1
- Key point 2

## Decisions
- Decision 1
- Decision 2

## Action Items
- [ ] Action 1 - Owner - Due Date
- [ ] Action 2 - Owner - Due Date

## Next Meeting
[Date/Time or TBD]
```

## Tone Guidelines

### Formal
- External communications
- Senior leadership
- Official requests
- Legal matters

### Professional
- Most work communications
- Cross-team collaboration
- Client communications
- Documentation

### Casual
- Team members you work with closely
- Informal updates
- Quick check-ins
- Chat messages

## Difficult Communications

### Delivering Bad News
1. Be direct but compassionate
2. Explain the situation clearly
3. Take responsibility where appropriate
4. Offer solutions or alternatives
5. End on a constructive note

### Giving Feedback
1. Be specific and objective
2. Focus on behavior, not personality
3. Provide examples
4. Suggest improvements
5. Offer support

### Handling Conflict
1. Stay calm and professional
2. Focus on facts
3. Listen to understand
4. Seek common ground
5. Propose solutions

## Communication Checklist

- [ ] Is the purpose clear?
- [ ] Is the right medium chosen?
- [ ] Is the tone appropriate?
- [ ] Is it concise enough?
- [ ] Are action items clear?
- [ ] Is timing considered?
- [ ] Proofread for errors?
